React to:

A

Action menu structure (Scenario 1)

Four sections: Update status / Reschedule / Replace / Remove. Status header at top shows current state. Most likely action ("Move to future") visually emphasized in blue. Pick if this organization works.

B

Drag-and-drop visual affordance (Scenario 2)

Source semester gets red "cannot drop on origin" border; target semester gets green border + drop hint; other valid semesters get lighter green; ghost tile follows cursor with rotation. The course's coreq-affected partner (CprE 1850) gets a yellow ring before drop completes.

C

Cascade preview content (Scenario 2)

Yellow callout listing all affected items: cascaded courses, semester credit changes (over/under thresholds). Three actions: Apply with cascade / Apply alone (with warning) / Cancel.

The drag-and-drop affordance and the cascade preview are decoupled — you'd see the same cascade preview whether you (a) clicked → "Move to future" → picked Sem 2, or (b) dragged the tile and dropped on Sem 2. One implementation, two entry points.
